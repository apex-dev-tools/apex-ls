Household Related Classes

Households in NPSP are implemented in two different ways. The first implementation in NPSP 2.0 provided a Household Object, that Contacts have an optional lookup field to. In NPSP 3.0, the Household Account model provided a different way to model households, using the Account object. Where possible, code is shared and provides support for both implementations. The Contact formula field HHId__c helps support this, by providing the Id of the Household Account or Household Object that the Contact is using.

The major areas of functionality in Households include:

  • Household Naming - automatic naming for the Household, Formal Greeting, and Informal Greeting
  • Manage Household UI - a javascript based ui for adding/removing Contacts from a Household.
  • Household Soft Credits - Opportunity Contact Roles created for Household members
  • Household Campaign Deduplication

The classes involved with Household Naming and Custom Naming include:

  • HH_HHObject_TDTM
  • HH_HouseholdNaming_BATCH
  • HH_HouseholdNaming_TEST
  • HH_HouseholdNaming
  • HH_Households_TDTM
  • HH_INaming
  • HH_NameSpec
  • HH_NameSpec_TEST

The classes involved with the Manage Household UI include:

  • HH_ManageHHAccount_TEST
  • HH_ManageHousehold_EXT
  • HH_ManageHousehold_TEST
  • the static resource javascript file ManageHousehold.js
  • the static resource css file ManageHousehold.css

The classes involved with Household Soft Credits include:

  • HH_OppContactRoles_TEST

The classes involved with Household Campaign Deduplication include:
